Recipe Book API — Документация

# Эндпоинты

| Метод   | Путь                   | Описание                                                      |
|:--------|:-----------------------|:--------------------------------------------------------------|
| `POST`  | `/recipes/`            | Добавить рецепт в базу данных                                 |
| `GET`   | `/recipes/`            | Получить список всех рецептов (в порядке убывания просмотров) |
| `GET`   | `/recipes/{recipe_id}` | Получить рецепт по ID + увеличить просмотры                   |


# POST `/recipes/` — Добавить рецепт в базу данных

Добавляет новый рецепт в базу данных.
Тело запроса (пример)

```json
{
  "name": "Борщ",
  "time": 60,
  "ingredients": "Свекла, капуста, морковь, лук, мясо, вода",
  "description": "Варить бульон 2 часа, добавить овощи, варить ещё 30 минут"
}
```

name:        str, обязательное поле, максимальная длина 100
time:        int, обязательное поле
ingredients: str, необязательно поле если незаполненно, то  = Null
description: str, необязательно поле, если незаполненно, то  = Null
views:       int, добавляется со значением 0

# GET /recipes/ — Получить список всех рецептов (в порядке убывания просмотров)

Возвращает список всех рецептов, отсортированный по количеству просмотров (от большего к меньшему).

Пример ответа (список словарей)

[
  {
    "id": 1,
    "name": "суп",
    "time": 60,
    "views": 21
  },
  {
    "id": 3,
    "name": "пюре",
    "time": 30,
    "views": 13
  },
  {
    "id": 2,
    "name": "борщ",
    "time": 70,
    "views": 1
  }
]

Если рецептов нет (пустой список)
Код 200

[]


# GET /recipes/{recipe_id} — Получить рецепт по ID + увеличить просмотры

Возвращает полную информацию о рецепте и автоматически увеличивает счетчик просмотров на 1.

Пример ответа (один объект)

{
  "id": 1,
  "name": "суп",
  "time": 60,
  "ingredients": "картошка, морковь",
  "description": "кинуть в кастрюлю"
}

Если по recipe_id ничего нет, то 
Код 404

{
  "detail": "Рецепт не найден"
}
